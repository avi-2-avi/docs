<Info>
  **Async (polling) · Câmera · R$ 1,10 por verificação · `confidence_score: 0.0–1.0`**
</Info>

## Endpoint

```
POST /v1/verify/face
GET  /v1/verify/:id
```

## Como funciona o liveness

Banana ID usa **passive liveness** — o provedor analisa a foto estática buscando sinais de spoofing: textura de pele, reflexo de tela, bordas de papel, profundidade estimada. Nenhuma ação do usuário é necessária além de tirar a selfie.

Isso significa que o arquivo enviado é uma **imagem** (`.jpg` ou `.png`), não um vídeo — menos fricção para o usuário final.

## Fluxo em 3 passos

Idêntico ao [Documento](/blocos/documento) — a diferença é o tipo de arquivo (imagem `.jpg`) e que o provedor compara o rosto com o documento enviado anteriormente.

### Passo 1 — criar a verificação

```bash
curl -X POST $BASE_URL/verify/face \
  -H "Authorization: Bearer bid_test_sandbox0000000000000000" \
  -H "Content-Type: application/json" \
  -d '{ "fixture": "test_face_match" }'
```

**Response `202`** — inclui `upload_url` e `upload_expires_at`.

### Passo 2 — enviar a selfie direto ao Storage

```bash
curl -X PUT "<upload_url>" \
  -H "Content-Type: image/jpeg" \
  --data-binary @selfie.jpg
```

### Passo 3 — polling até o resultado

```bash
curl $BASE_URL/verify/ver_8f2a1c9b \
  -H "Authorization: Bearer bid_test_sandbox0000000000000000"
```

Resultado em ~4s após o upload. Response final inclui `evidence_url` (a selfie para revisão manual se necessário).

## Códigos de resultado

| `reason` | Significado | `status` | `reason_group` |
|---|---|---|---|
| `null` | Match facial + liveness confirmados | `approved` | `null` |
| `face_low_quality` | Iluminação / foco insuficiente | `review` | `review` |
| `face_no_match` | Rosto não corresponde ao documento | `rejected` | `block` |
| `face_no_liveness` | Não detectou pessoa viva (foto/tela) | `rejected` | `fraud` |
| `face_photo_of_photo` | Foto de foto | `rejected` | `fraud` |

<Warning>
  `face_no_liveness` e `face_photo_of_photo` têm `reason_group: fraud` — não retentar automaticamente. Escale para revisão ou bloqueie.
</Warning>

## Exemplo Node.js completo

```js
const BASE = process.env.BASE_URL; // Dashboard → Configurações → API
const KEY  = "bid_test_sandbox0000000000000000";

// 1. Criar
const { id, upload_url } = await fetch(`${BASE}/verify/face`, {
  method: "POST",
  headers: { "Authorization": `Bearer ${KEY}`, "Content-Type": "application/json" },
  body: JSON.stringify({ fixture: "test_face_no_live" }),
}).then(r => r.json());

// 2. Upload da selfie
await fetch(upload_url, {
  method: "PUT",
  headers: { "Content-Type": "image/jpeg" },
  body: imageBuffer,
});

// 3. Polling
let result;
while (true) {
  result = await fetch(`${BASE}/verify/${id}`, {
    headers: { "Authorization": `Bearer ${KEY}` },
  }).then(r => r.json());
  if (result.status !== "processing") break;
  await new Promise(r => setTimeout(r, 2000));
}

// result.status       === "rejected"
// result.reason       === "face_no_liveness"
// result.reason_group === "fraud"
```
