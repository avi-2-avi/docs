<Info>
  **Async (polling) · Foto RG/CNH · R$ 0,80 por verificação · `confidence_score: 0.0–1.0`**
</Info>

## Endpoint

```
POST /v1/verify/document
GET  /v1/verify/:id
```

## Fluxo em 3 passos

### Passo 1 — criar a verificação

```bash
curl -X POST $BASE_URL/verify/document \
  -H "Authorization: Bearer bid_test_sandbox0000000000000000" \
  -H "Content-Type: application/json" \
  -d '{ "fixture": "test_doc_blurry" }'
```

**Response `202`:**

```json
{
  "id": "ver_3d77e0a1",
  "type": "document",
  "status": "processing",
  "confidence_score": null,
  "reason": null,
  "reason_group": null,
  "upload_url": "https://…supabase.co/storage/v1/object/upload/sign/…?token=…",
  "upload_expires_at": "2026-06-20T14:40:00Z",
  "created_at": "2026-06-20T14:30:00Z"
}
```

### Passo 2 — enviar o arquivo direto ao Storage

```bash
curl -X PUT "<upload_url>" \
  -H "Content-Type: image/jpeg" \
  --data-binary @documento.jpg
```

O arquivo vai **direto a um bucket privado** — sem passar pela Edge Function. O `upload_url` expira em ~10 minutos.

<Warning>
  Não envie o arquivo pela sua API — use o `PUT` direto para o `upload_url`. Isso garante que o arquivo nunca transite pelo seu servidor.
</Warning>

### Passo 3 — polling até o resultado

```bash
curl $BASE_URL/verify/ver_3d77e0a1 \
  -H "Authorization: Bearer bid_test_sandbox0000000000000000"
```

Faça polling a cada ~2s. O resultado aparece em ~4s após o upload.

**Response final:**

```json
{
  "id": "ver_3d77e0a1",
  "type": "document",
  "status": "review",
  "confidence_score": 0.61,
  "reason": "doc_blurry",
  "reason_group": "review",
  "evidence_url": "https://…supabase.co/storage/…?token=…",
  "created_at": "2026-06-20T14:30:00Z"
}
```

O `evidence_url` é uma URL assinada (GET, expira em ~5 min) para visualizar o arquivo na Fila de revisão.

## Códigos de resultado

| `reason` | Significado | `status` | `reason_group` |
|---|---|---|---|
| `null` | Documento válido e autêntico | `approved` | `null` |
| `doc_blurry` | Imagem desfocada | `review` | `review` |
| `doc_glare` | Reflexo / iluminação ruim | `review` | `review` |
| `doc_expired` | Documento vencido | `rejected` | `block` |
| `doc_unregistered` | Não confere com base gov. | `rejected` | `block` |
| `doc_photocopy` | É fotocópia / foto de foto | `rejected` | `fraud` |
| `doc_tampered` | Indícios de adulteração | `rejected` | `fraud` |

## Exemplo Node.js completo

```js
const BASE = process.env.BASE_URL; // Dashboard → Configurações → API
const KEY  = "bid_test_sandbox0000000000000000";
const headers = { "Authorization": `Bearer ${KEY}`, "Content-Type": "application/json" };

// 1. Criar
const { id, upload_url } = await fetch(`${BASE}/verify/document`, {
  method: "POST", headers,
  body: JSON.stringify({ fixture: "test_doc_blurry" }),
}).then(r => r.json());

// 2. Upload
await fetch(upload_url, {
  method: "PUT",
  headers: { "Content-Type": "image/jpeg" },
  body: fileBuffer, // Buffer ou ReadableStream
});

// 3. Polling
let result;
while (true) {
  result = await fetch(`${BASE}/verify/${id}`, {
    headers: { "Authorization": `Bearer ${KEY}` },
  }).then(r => r.json());
  if (result.status !== "processing") break;
  await new Promise(r => setTimeout(r, 2000));
}

console.log(result.status);       // "review"
console.log(result.reason_group); // "review"
console.log(result.evidence_url); // URL assinada para ver a imagem
```

## Quando usar

Antes de liberar crédito ou acesso a recursos de alto valor. O `confidence_score` te permite ajustar o limiar de aprovação automática nas [Configurações](/referencia/objeto-verificacao#confidence_score).
