## 1. Crie sua conta

[Crie uma conta gratuita](https://banana-id.lovable.app/signup). Você cai direto no sandbox — grátis, ilimitado, sem cartão de crédito.

## 2. Pegue sua chave de teste

No [Dashboard → Chaves de API](https://banana-id.lovable.app/app/keys), copie sua chave `bid_test_…`. Ela fica no header de todas as chamadas:

```
Authorization: Bearer bid_test_sua_chave_aqui
```

## 3. Faça sua primeira verificação

<Tabs>
  <Tab title="CPF (sync)">
    ```bash
    curl -X POST https://zjuftegykhuempxnkikx.supabase.co/functions/v1/v1/verify/cpf \
      -H "Authorization: Bearer bid_test_sandbox0000000000000000" \
      -H "Content-Type: application/json" \
      -d '{ "cpf": "390.533.447-05" }'
    ```

    Resposta imediata:

    ```json
    {
      "id": "ver_70464409",
      "type": "cpf",
      "status": "approved",
      "confidence_score": null,
      "reason": null,
      "reason_group": null,
      "created_at": "2026-06-20T17:35:50Z"
    }
    ```
  </Tab>
  <Tab title="Node.js">
    ```js
    const res = await fetch(
      "https://zjuftegykhuempxnkikx.supabase.co/functions/v1/v1/verify/cpf",
      {
        method: "POST",
        headers: {
          "Authorization": "Bearer bid_test_sandbox0000000000000000",
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ cpf: "390.533.447-05" }),
      }
    );
    const verification = await res.json();
    console.log(verification.status); // "approved"
    ```
  </Tab>
</Tabs>

## 4. Teste outros cenários

Use fixtures para forçar qualquer resultado sem dados reais:

```bash
# CPF inativo
curl ... -d '{ "cpf": "test_cpf_inactive" }'
# → status: "rejected", reason: "cpf_inactive", reason_group: "block"

# CPF em lista de fraude
curl ... -d '{ "cpf": "test_cpf_fraud" }'
# → status: "rejected", reason: "cpf_fraud_list", reason_group: "fraud"
```

Veja a lista completa de fixtures em [Sandbox e dados de teste](/sandbox).

## 5. Lide com os três resultados

```js
if (verification.status === "approved") {
  // prosseguir
} else if (verification.status === "review") {
  // enviar para fila de revisão manual
  // reason_group === "review" → aguardar análise
} else {
  // status === "rejected"
  if (verification.reason_group === "fraud") {
    // bloquear, não reintentar
  } else if (verification.reason_group === "retry") {
    // pedir novo envio ao usuário
  } else {
    // reason_group === "block" → validação alternativa
  }
}
```

<Tip>
  Use `reason_group` para decidir o próximo passo — sem precisar interpretar o enum de `reason` completo. Veja a [tabela de reason_group](/referencia/reasons).
</Tip>
