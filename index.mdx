## Como funciona

Banana ID **não implementa biometria** — encapsula um provedor existente e expõe um contrato mais simples e acionável. Todo resultado tem o mesmo shape:

```json
{
  "id": "ver_8f2a1c9b",
  "type": "face",
  "status": "approved",
  "confidence_score": 0.94,
  "reason": null,
  "reason_group": null,
  "created_at": "2026-06-18T14:32:07Z"
}
```

`status` é sempre `approved`, `rejected` ou `review`. `reason_group` (`retry` / `review` / `block` / `fraud`) diz o que fazer sem precisar interpretar o enum completo.
