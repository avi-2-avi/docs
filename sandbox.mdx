## Como funciona o sandbox

Use uma chave `bid_test_…` e passe um **fixture** no campo de entrada. O fixture determina o resultado de forma determinista — sem análise real, sem PII, sem cobrança.

Sem fixture → `approved` por padrão (caminho feliz).

## Convenção por nome de arquivo

Para os blocos async (documento/face), se não passar o campo `fixture`, o **nome do arquivo enviado** serve de fixture:

```
test_doc_blurry.jpg   → doc_blurry
test_face_no_live.mp4 → face_no_liveness
```

## O arquivo é real, o veredicto é simulado

Mesmo no sandbox, o arquivo que você envia vai para o Storage privado e aparece como evidência real na Fila de revisão (`evidence_url`). O que é simulado é o **veredicto** — que é determinado pelo fixture, não pelo conteúdo do pixel.

Isso significa que você integra de verdade — autentica, faz upload, faz polling — e vê a evidência na fila. Só o resultado é previsível.
