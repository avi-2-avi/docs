## Como usar

`reason_group` é o campo principal para decisões no seu código. `reason` dá o detalhe para logs e para exibir ao seu time de revisão.

```js
if (v.reason_group === "fraud")  { block(); return; }
if (v.reason_group === "block")  { requestAlternative(); return; }
if (v.reason_group === "review") { sendToQueue(v); return; }
if (v.reason_group === "retry")  { askUserToRetry(); return; }
```

## CPF

| `reason` | Significado | `status` | `reason_group` |
|---|---|---|---|
| `cpf_not_found` | CPF não existe na base | `rejected` | `block` |
| `cpf_inactive` | CPF existe mas está irregular | `rejected` | `block` |
| `cpf_fraud_list` | CPF em lista de fraude | `rejected` | `fraud` |
| `cpf_invalid_format` | Formato inválido (erro de input) | `rejected` | `retry` |

## Por que um enum reduzido?

O provedor que encapsulamos tem dezenas de códigos internos. Banana ID os mapeia para este conjunto estável e documentado — assim seu código não quebra quando o provedor muda sua nomenclatura interna.
